# 全球空气质量指数（AQI）交互式可视化 - 课程大作业

## 项目概述

本项目是一个交互式全球空气质量指数（AQI）可视化工具，帮助用户直观探索全球空气污染的空间分布与时间演变趋势。工具以等值线地图为核心，集成时间轴刷选、城市详情查看、多城市对比和国家/地区过滤等交互功能，采用 **Vue 3 + D3.js** 技术栈构建，并部署至 GitHub Pages。

**在线访问地址**: https://[your-username].github.io/global-aqi-viz/

---

## 核心功能

### 1. 交互式地图探索
- **缩放/平移**: 鼠标滚轮缩放，拖拽平移，支持0.5x-8x缩放范围
- **城市气泡**: 颜色代表AQI等级，半径与AQI值成正比（双重编码）
- **智能标签**: 为污染较严重的城市自动显示名称标签
- **点击交互**: 左键点击城市查看污染物时间序列
- **对比功能**: 右键点击城市加入对比，双城市趋势并排显示

### 2. 时间轴动态查询
- **刷选器**: 拖动时间范围手柄，实时过滤数据
- **播放功能**: 自动推进时间，观察污染演变过程
- **Sparkline趋势**: 时间轴下方显示平均AQI趋势指示器

### 3. 多视图协调
- **地图 → 折线图**: 点击城市触发详情视图
- **时间刷选 → 地图/折线图**: 时间范围变化联动所有视图
- **国家过滤 → 全局**: 过滤器同步更新地图和图表

### 4. 污染物详情分析
- **多污染物切换**: PM2.5、PM10、NO₂、O₃ 图例点击显示/隐藏
- **WHO参考线**: 折线图上标注WHO空气质量指导值
- **数据点悬停**: 鼠标悬停显示具体数值和日期
- **双城市对比**: 实线/虚线区分主城市和对比城市

## 设计说明

### 可视化方案旨在解答的问题

1. **空间分布问题**: 全球不同地区的空气质量指数（AQI）存在怎样的空间分布差异？哪些区域污染最严重、哪些区域空气质量较好？

2. **时间趋势问题**: 选定区域（国家或城市）的AQI及主要污染物浓度在时间上是如何变化的？是否存在季节性模式或改善/恶化趋势？

3. **污染物结构问题**: 不同城市的污染"成分"有何不同？PM2.5、PM10、NO₂等污染物在各地AQI中扮演的角色如何？

### 设计决策依据

#### 可视化编码选择

| 数据维度 | 视觉编码 | 理由 |
|----------|----------|------|
| AQI值 | 颜色（色相/饱和度） | 颜色是最直观表示等级/数值的通道，用户可快速识别高污染区域 |
| 城市地理位置 | 二维空间位置（x, y） | 地图投影天然适合展示地理分布模式 |
| AQI相对大小 | 气泡半径 | 辅助编码，增强城市间的视觉对比 |
| 时间 | 水平轴位置 | 时间序列的标准编码，用户习惯 |
| 污染物浓度 | 垂直轴位置 + 线条颜色 | 多曲线折线图清晰对比不同污染物的变化趋势 |
| 国家归属 | 过滤器选择 | 非直接视觉编码，通过交互控制显示范围 |

#### 交互技术选择

| 交互技术 | 应用场景 | 替代方案考量 |
|----------|----------|-------------|
| **缩放+平移** | 地图空间探索 | 替代：固定视图+区域切换。选择缩放平移是因为它更符合"概览→细节"的探索范式 |
| **刷选（Brushing）** | 时间范围选择 | 替代：下拉选择年份/月份。选择刷选是因为它更直观、支持连续范围选择 |
| **按需显示细节（点击+折线图）** | 城市详情查看 | 替代：悬浮面板。选择点击触发折线图是因为折线图能提供更丰富的时间趋势信息 |
| **过滤器** | 国家/地区聚焦 | 替代：地图高亮。选择过滤器是为了简化视图、聚焦特定区域 |

#### 考虑过但未采用的替代方案

1. **热力图代替等值区域着色**: 热力图更适合高密度站点数据，但全球城市数据分布稀疏，等值区域着色更能传达整体空间模式。

2. **3D地球代替2D地图**: 3D更炫酷但交互复杂度高，且2D平面地图的墨卡托投影对全球尺度的AQI对比已足够有效。

3. **React代替Vue**: D3与React的集成同样成熟，但Vue的模板语法更直观简洁，学习曲线更平缓，适合三周的开发周期。

4. **Leaflet代替D3绘制地图**: Leaflet在瓦片地图加载和移动端体验上更优，但D3在自定义数据驱动的视觉编码上更灵活。

## 数据源

### 主要数据源

- **OpenAQ**: 世界最大的开源空气质量数据平台，提供超过20亿条来自22,500+个数据源的测量数据，覆盖142个国家
  - 网站: https://openaq.org/
  
- **World Atlas (TopoJSON)**: 世界地图边界数据
  - CDN: https://cdn.jsdelivr.net/npm/world-atlas@2/land-110m.json

### 数据结构

预处理后的静态JSON文件包含两个主要部分：

**cities.json** - 城市元数据:
```json
{
  "id": "beijing",
  "name": "北京",
  "country": "中国",
  "lat": 39.9042,
  "lng": 116.4074
}
```

**measurements.json** - 月度测量数据:
```json
{
  "city_id": "beijing",
  "date": "2024-01",
  "aqi": 152,
  "pm25": 115.2,
  "pm10": 140.5,
  "no2": 48.3,
  "o3": 62.1,
  "category": "unhealthy"
}
```

### AQI等级定义与颜色编码

| AQI范围 | 等级 | 颜色 | 色值 |
|---------|------|------|------|
| 0-50 | 优（Good） | 绿色 | #00E400 |
| 51-100 | 良（Moderate） | 黄色 | #FFFF00 |
| 101-150 | 轻度污染（Unhealthy for Sensitive） | 橙色 | #FF7E00 |
| 151-200 | 中度污染（Unhealthy） | 红色 | #FF0000 |
| 201-300 | 重度污染（Very Unhealthy） | 紫色 | #8F3F97 |
| 300+ | 严重污染（Hazardous） | 褐红色 | #7E0023 |

### Vue与D3的分工原则

- **Vue**: 管理组件生命周期、响应式数据和UI布局
- **D3**: 负责地理投影、数据到视觉元素的映射、缩放平移变换以及动态图表绘制
- **桥梁**: 通过`watch`和`computed`建立响应式状态与D3重绘的连接

### 耗时最多的环节

1. **地图绘制与数据绑定（10小时）**: D3的地图投影、GeoJSON加载、城市坐标映射涉及较多细节调优，特别是确保经纬度投影后视觉比例合适，以及城市气泡与地图的相对定位准确性。

2. **交互联动（8小时）**: 时间轴刷选与多视图联动是交互核心，涉及Vue响应式状态与D3重绘的协调，需要仔细处理数据流和事件传递。

3. **折线图与城市点击联动（8小时）**: D3折线图的绘制、坐标轴设置、多曲线显示以及点击事件的联动逻辑较为复杂，尤其是图例切换、数据点悬停提示等细节功能的打磨。

### 主要挑战与解决方案

**挑战1: Vue与D3的分工边界**
- **问题**: 初期不清楚哪些逻辑应该放在Vue，哪些应该放在D3
- **解决**: 确立原则——Vue管理状态和布局，D3负责数据驱动的视觉绘制，通过`watch`和`computed`建立桥梁

**挑战2: TopoJSON转换**
- **问题**: 世界地图数据是TopoJSON格式，需要转换为GeoJSON
- **解决**: 在`index.html`中引入topojson-client CDN，使用`topojson.feature()`进行转换

**挑战3: 响应式重绘性能**
- **问题**: 频繁的地图重绘导致性能问题
- **解决**: 使用`ResizeObserver`优化窗口调整，添加动画过渡提升用户体验

## 高级功能亮点（加分项）

### 1. 多城市对比
- **实现**: 右键点击城市可加入对比，折线图以虚线显示第二城市数据
- **价值**: 支持跨城市空气质量对比分析

### 2. 时间轴播放
- **实现**: 点击播放按钮，时间窗口自动推进，地图和图表实时更新
- **价值**: 动态展示污染演变过程，增强探索体验

### 3. Sparkline趋势指示器
- **实现**: 时间轴下方显示全局平均AQI趋势，带渐变填充
- **价值**: 快速感知整体污染变化趋势

### 4. WHO指南参考线
- **实现**: 折线图上绘制WHO空气质量指导值的虚线
- **价值**: 提供健康参考标准，增强数据解读

### 5. 智能标签显示
- **实现**: 仅为AQI > 80的城市显示名称标签
- **价值**: 避免视觉混乱，突出关注点

### 6. 动画过渡效果
- **实现**: 城市气泡展开动画、折线图平滑曲线（MonotoneX）
- **价值**: 提升视觉体验，引导用户注意力
